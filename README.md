# Admission Analytics Dashboard & Reporting Project

## Project Overview
This project was developed as part of a Data Analytics & Visualization Internship.  
The objective of the project was to transform fragmented admission data into a structured and interactive dashboard that supports data-driven decision-making.

The project focused on:
- Data cleaning and preprocessing
- Dashboard development
- KPI monitoring
- Trend analysis
- Insight generation
- Interactive reporting

---

# Objective
The main objective of this project was to analyze the admission process by tracking:
- Application performance
- Conversion rates
- Geographic distribution
- Program-wise demand
- Admission trends
- Recruitment effectiveness

The dashboard was designed to provide clear and actionable insights for better admission and recruitment analysis.

# Data Cleaning & Preprocessing
The raw dataset contained:
- Missing values
- Duplicate records
- Redundant columns
- Inconsistent formatting
- Timestamp formatting issues

### Data Cleaning Steps
- Removed completely null columns
- Removed redundant merged columns
- Standardized text values (Country, City)
- Standardized phone number formats
- Converted timestamps into readable date-time format
- Standardized date formats
- Removed duplicate entries
- Standardized missing value placeholders

# Key Insights

### Program Distribution
- Business Analytics and Computer Science received the highest number of applications.
- Programs like HCI showed comparatively lower application volumes.

### Geographic Distribution
- Most applications originated from Asia and Europe.
- Lower application volumes were observed from Africa and South America.

### Admission Trends
- Application activity peaked during January–February and September–October admission cycles.

### College-wise Distribution
- A significant share of applications came from a small number of major colleges.
